UI/UX DESIGN UI/UX 디자인

With coding works, I always feel there needs to be a good balance between user interface and user experience. ACCESSIBILITY 접근성

I write HTML5 code with accessibility in mind so I keep my code as semantic as possible. For example, alternative text for an image and many other features. HTML5 코드를 쓸 때 접근성을 생각하며 최대한 시맨틱한 코드를 쓰는 편입니다. 예를 들어, 이미지를 위한 alternative 텍스트를 쓰는 것입니다.

TESTING 테스트

As ISTQB Foundation Level Certified Tester I review my code in two phases. Firstly, I write a code and finish it from a developer's perspective. After this I review my code from a tester's perspective with a purpose to find any bugs in my code and fix them before submitting my work. Having two perspectives keeps my code as clean as possible. ISTQB Foundation Level Certified 테스터로서 저는 두 단계에 걸쳐 코드를 확인합니다. 우선, 코드를 개발자의 관점에서 작성합니다. 그 이후 테스터의 관점에서 코드를 확인하여 버그를 수정하곤 합니다. 두 가지 관점을 가지고 업무를 진행하면 코드를 가능한 한 깨끗하게 유지할 수 있습니다.
